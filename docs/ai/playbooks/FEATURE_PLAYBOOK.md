# Feature Delivery Playbook

## Discovery

- user goal
- MVP behavior
- out-of-scope behavior
- data/API needs
- permissions/security
- empty/error/loading states
- analytics/monitoring needs
- App Store implications

## Design

- architecture flow
- model/API contract
- state owner
- migration
- backward compatibility
- test plan
- rollout/rollback

## Delivery gates

- feature works
- existing behavior preserved
- tests added
- analyzer/build pass
- runtime QA
- documentation updated
- no hidden placeholder or mock in production
